# User Story #3: «Есть ли у меня платежи на этой неделе?»

## Контекст

- Текущая дата (для примера): **2026-07-15** (среда)
- Текущая неделя: **понедельник 2026-07-13 .. воскресенье 2026-07-19**

## Ожидаемый ReAct-трейс

```
============================================================
[Thought]
Нужны все обязательства. Определяю "на этой неделе" как
понедельник-воскресенье относительно текущей даты 2026-07-15.
Окно: 2026-07-13 .. 2026-07-19.
============================================================

============================================================
[Action]
tool=get_obligations
input={"status": null, "category": null}
============================================================

============================================================
[Observation]
[12 записей с next_payment_date от 2026-07-16 до 2026-08-05]
============================================================

--- ANSWER ---
Да, на этой неделе (2026-07-13 .. 2026-07-19) у вас 4 платежа:

| Дата       | Название        | Сумма      |
|------------|-----------------|-----------:|
| 2026-07-16 | Ростелеком      | 800.00 RUB |
| 2026-07-17 | Яндекс.Плюс     | 299.00 RUB |
| 2026-07-18 | Spotify Premium | 5.99 USD   |
| 2026-07-19 | Мосэнерго       | 1500.00 RUB|

Итого на неделе: 2599.00 RUB + 5.99 USD.
```

## Что демонстрирует

1. **Определение "этой недели"** — агент сам вычисляет понедельник-воскресенье.
2. **Фильтрация по дате** — только записи в окне.
3. **Без агрегации по валютам** — если пользователь не просил конвертацию,
   агент не делает лишних вызовов API (это видно по отсутствию convert_currency
   в трейсе).
4. **Чёткий ответ** — "Да, 4 платежа" + список.
